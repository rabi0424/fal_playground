# スマホ対応 UI 設計書（iPhone 16）

fal playground を iPhone 16 で快適に使えるようにするための UI 設計。
本書は「どのようなデザインにするか」の決定事項をまとめたもの。実装は別途行う。

## 1. 対象デバイスの前提

| 項目 | 値 |
|---|---|
| 機種 | iPhone 16（6.1 インチ） |
| 論理解像度 | 393 × 852 pt（@3x、物理 1179 × 2556 px） |
| ブラウザ | iOS Safari（Chrome for iOS も同じ WebKit エンジン） |
| 特記事項 | Dynamic Island（上部）、ホームインジケータ（下部）、ノッチ相当のセーフエリアあり |

設計上のブレークポイントは **430px 以下をモバイル扱い** とする
（iPhone 16 = 393pt、16 Plus / Pro Max = 430pt までを同一レイアウトでカバー。
既存の 900px ブレークポイントは「1 カラム化」用としてそのまま残す）。

## 2. 現状の課題

現在は `@media (max-width: 900px)` で 2 カラム → 1 カラムに落とすだけの簡易対応。
iPhone 16（幅 393pt）では以下の問題が起きる。

1. **生成ボタンが埋もれる**
   コントロール（モデル / プロンプト / LoRA / サイズ / 詳細設定）が縦に長く、
   その最下部に生成ボタン、さらに下に結果・ギャラリーが続く。
   「プロンプトを直して再生成 → 結果を見る」の往復スクロールが大きい。
2. **フォーカス時の自動ズーム**
   iOS Safari は font-size が 16px 未満の input / textarea / select にフォーカスすると
   ページ全体をズームする。現状は 14px なので毎回ズームが発生し、操作感を大きく損なう。
3. **セーフエリア未対応**
   `viewport-fit=cover` も `env(safe-area-inset-*)` も未使用。
   下部固定要素を置くとホームインジケータと重なる。
4. **タッチターゲット不足**
   `ghost-btn.small`（padding 4px 10px、12px 文字）や
   ギャラリー内の削除ボタンが Apple HIG 推奨の 44pt 四方に満たない。
5. **ライトボックスがキーボード前提**
   画像切替が ←/→ キーのみでスワイプ不可。閉じる操作も「画像外をタップ」だけで
   分かりにくい。
6. **hover 依存の UI**
   `.gallery-item:hover` などタッチでは発火しない装飾があり、押せる要素が判別しづらい。
7. **100vh 問題**
   `max-height: calc(100vh - 48px)` や `max-height: 60vh` は iOS Safari の
   アドレスバー伸縮で実際の可視領域とズレる。

## 3. デザイン決定

### 3.1 全体方針

- **フレームワークは導入しない。** 既存の「静的 3 ファイル構成・ビルド不要」を維持し、
  CSS のメディアクエリ＋少量の JS 追加のみで対応する。
- **レイアウト形式は「1 カラム縦積み ＋ 下部固定の生成バー」に決定。**
  タブ切替（生成 / ギャラリー）やボトムシート案も検討したが、
  - タブ切替: 生成結果を見ながらプロンプトを直す往復が増える
  - ボトムシート: 実装コストが高く、既存のシンプルな構成に合わない
  ため不採用。縦積みなら既存 DOM をほぼ変えずに済み、
  「下部固定バー」で最重要操作（生成）への到達性を確保できる。

### 3.2 画面構成（430px 以下）

```
┌─────────────────────────┐
│ ⬤ Dynamic Island        │ ← safe-area-inset-top
├─────────────────────────┤
│ fal playground   自動 🔑 │ ← トップバー（スクロールで流れる・固定しない）
├─────────────────────────┤
│ プロンプト                │ ← 最上部に昇格（最も触る要素）
│ ┌─────────────────────┐ │
│ │ textarea            │ │
│ └─────────────────────┘ │
│ モデル [FLUX.1 schnell ▾]│
│ サイズ [横長16:9▾] 枚数[1]│
│ ▸ LoRA                  │ ← アコーディオン（details）に集約
│ ▸ 詳細設定（シード等）     │ ← 既存の details を流用
├─────────────────────────┤
│ 生成結果（detail）        │
│ ┌─────────────────────┐ │
│ │      画像            │ │
│ └─────────────────────┘ │
│ [開く] [保存] seed:1234  │
├─────────────────────────┤
│ ギャラリー      すべて削除 │
│ ┌───┐ ┌───┐ ┌───┐      │ ← 3 列グリッド
│ └───┘ └───┘ └───┘      │
├─────────────────────────┤
│ ステータス/エラー          │
│ ┌─────────────────────┐ │
│ │      生成する         │ │ ← 下部固定バー（常時表示）
│ └─────────────────────┘ │
│ ━━━━ home indicator ━━━ │ ← safe-area-inset-bottom 分を確保
└─────────────────────────┘
```

決定事項:

1. **下部固定の生成バー**
   - `position: fixed; bottom: 0` のバーに生成ボタンを移し、常時タップ可能にする
     （モバイル時のみ。PC ではフォーム内の現在位置のまま）。
   - バーの padding-bottom に `env(safe-area-inset-bottom)` を加算し、
     ホームインジケータと重ねない。
   - 生成中のステータス / エラーはバー直上に重ねて表示し、
     スクロール位置に関わらず進捗が見えるようにする。
   - 本文最下部にはバーの高さぶんの余白を確保し、コンテンツが隠れないようにする。
2. **フォームの整理（モバイルのみ）**
   - プロンプトをフォーム先頭に移動（最頻操作を最短距離に）。
   - モデル・サイズ・枚数は 1〜2 行にまとめて露出したままにする。
   - LoRA 関連（LoRA リスト / 比較モード）は `<details>` アコーディオンに畳む。
     詳細設定（シード / ステップ / ガイダンス）は既存の `<details>` のまま。
   - DOM の並び替えは CSS `order`（controls を flex にしているため）で行い、
     PC レイアウトには影響させない。
3. **結果表示**
   - 生成完了時に detail パネルへ自動スクロールする（モバイルのみ）。
   - 画像は幅 100%・`max-height: 70svh`（svh 単位に変更）。

### 3.3 iOS Safari 対応（必須事項）

| 項目 | 決定 |
|---|---|
| viewport | `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` に変更 |
| 自動ズーム防止 | input / select / textarea の font-size を **16px** にする（モバイルのみでよいが、全体 16px 化しても崩れないため一律 16px とする） |
| セーフエリア | 下部固定バー・トップバー・ライトボックスに `env(safe-area-inset-*)` を適用 |
| ビューポート単位 | `100vh` / `60vh` / `70vh` → `svh`（安定表示）または `dvh` に置換 |
| タップハイライト | 主要ボタンに `-webkit-tap-highlight-color: transparent` ＋ `:active` スタイルで押下フィードバック |
| スクロール | ギャラリーの内部スクロールはモバイルでは廃止し、ページスクロールに統一（入れ子スクロールは誤操作の元） |

### 3.4 タッチターゲット / タイポグラフィ

- ボタン類の最小ヒットエリアを **44 × 44pt** にする。
  - `ghost-btn.small` はモバイルでは padding 10px 14px / font-size 14px に拡大。
  - `primary-btn`（生成）は高さ 50pt・font-size 17px（iOS 標準のボタン文字サイズ）。
  - アイコンボタン（⇄）は 44 × 44pt に拡大。
- 本文 font-size はモバイル 15px、入力系は 16px（ズーム防止のため厳守）。
- 行間 1.6 は維持。

### 3.5 ギャラリー（モバイル）

- グリッドは `repeat(3, 1fr)`（393pt ÷ 3 ≒ 125pt/枚で視認性十分）。
- サムネ下のプロンプト 2 行表示・モデル名は維持するが、
  **「削除」ボタンはカード上から撤去**し、詳細表示（detail）側の操作列に移す。
  カード上の小さい削除ボタンは誤タップリスクが高いため。
- hover 装飾の代わりに `:active` で border 色を変える。
- 選択中カードの `box-shadow` リングは維持（視認性が高いため）。

### 3.6 ライトボックス（モバイル）

- **横スワイプで前後の画像へ移動**（touchstart / touchend の 40px 以上の水平移動で判定。
  既存の `lightboxNav()` を呼ぶだけなので実装は小さい）。
- 右上に明示的な「✕ 閉じる」ボタンを追加（44pt、`safe-area-inset-top` 考慮)。
- 複数枚あるときは下部に「2 / 4」形式のページ表示を出す。
- 背景タップで閉じる挙動は維持。

### 3.7 キーボードショートカットの代替

- Cmd/Ctrl + Enter は物理キーボードがないと使えないが、
  下部固定バーの生成ボタンが常時見えているため代替 UI は不要と判断。
- ←/→ でのギャラリー移動は、ライトボックスのスワイプ対応で代替。

### 3.8 変更しないこと

- カラーパレット・ダークモード（auto / light / dark）は現行のまま。
  iOS のシステム外観設定に `prefers-color-scheme` で追従する現仕様は
  そのままスマホでも正しく機能する。
- API キーダイアログは `width: min(420px, 90vw)` で既に収まるため、
  input の 16px 化以外は変更しない。
- PC（431px 以上）のレイアウト・挙動は一切変更しない。

## 4. 実装方針（次フェーズ）

1. `index.html`: viewport メタ変更、（必要なら）ライトボックスの閉じるボタン追加
2. `style.css`: `@media (max-width: 430px)` ブロックを追加し、3 章の内容を反映。
   vh → svh 置換と入力 16px 化はメディアクエリ外の共通変更
3. `app.js`: ライトボックスのスワイプ処理、生成完了時の自動スクロールを追加
   （いずれも 30 行程度の追加で済む見込み）

既存ファイル 3 つへの追記のみで完結し、ビルド工程・依存追加は発生しない。

## 5. 検証方法

- iOS Safari 実機（iPhone 16）で確認。LAN 配信（README の手順）で実機アクセス可能
- 実機がない場合は Safari の レスポンシブ・デザイン・モード、
  または Chrome DevTools の iPhone 16 プリセット（393 × 852）で確認
- チェック項目:
  - [ ] 入力フォーカスでページがズームしない
  - [ ] 生成ボタンが常に見えていて、ホームインジケータと重ならない
  - [ ] 生成中のステータスがスクロール位置に関わらず見える
  - [ ] ライトボックスがスワイプで切り替わり、✕ で閉じられる
  - [ ] ダーク/ライト両テーマで表示が破綻しない
  - [ ] 横向き（852 × 393）でも操作不能にならない
